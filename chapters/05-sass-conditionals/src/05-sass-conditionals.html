Sass · Conditionals

Sass Conditionals

@if / @else if / @else let you branch at compile time — perfect for variant APIs and guardrails.

Compile-time branching

Sass conditionals run at compile time. They decide which declarations are emitted into the final CSS. The browser never evaluates these branches — it only sees the result.

  • Conditionals are ideal for variant APIs (tone, size, intent) built on top of shared base styles.
  • Each variant is just a different set of arguments passed into the same mixin.
  • Invalid inputs can be rejected early using @error, before any CSS is generated.

No JavaScript is involved. Variants are classes, and the branching happens once during compilation.

Conditional mixin

@mixin button($intent) {
  @if $intent == "primary" {
    background: var(--accent);
    color: white;
  } @else if $intent == "danger" {
    background: var(--danger);
    color: white;
  } @else {
    background: var(--surface);
    color: var(--ink);
  }
}

Variants using the same mixin

.btn--primary { @include button("primary"); }
.btn--danger  { @include button("danger"); }
.btn--ghost   { @include button("ghost"); }

Component Variations

This demo uses a single mixin that accepts tone and size. Inside the mixin, @if branches choose which declarations get emitted.

  • Tone decides background.
  • Size decides padding and font-size.
  • Guardrails: unknown values fail fast with @error.

Open partials/_badge.scss and trace the two branch points: tone first, size second.

Mixin Component Variation

@mixin badge($tone: cool, $size: md) {
  display: inline-flex;
  align-items: center;
  gap: $space-2;
  border-radius: $radius-pill;
  border: 1px solid $stroke;
  letter-spacing: 0.02em;
  color: $ink;

  // size (compile-time)
  @if $size == sm {
    padding: 0.25rem $space-3;
    font-size: 0.85rem;
  } @else if $size == lg {
    padding: $space-2 $space-4;
    font-size: 1.05rem;
  } @else if $size == md {
    padding: 0.35rem $space-4;
    font-size: 0.95rem;
  } @else {
    @error "Unknown badge size: #{$size}. Use sm|md|lg.";
  }

  // tone (compile-time)
  @if $tone == hot {
    background: $hot;
  } @else if $tone == ghost {
    background: transparent;
  } @else if $tone == cool {
    background: $cool;
  } @else {
    @error "Unknown badge tone: #{$tone}. Use cool|hot|ghost.";
  }
}

What to inspect

  1. Inspect .badge--hot-lg (it’s visually distinct).
  2. Find the declarations coming from the badge mixin.
  3. Jump to partials/_badge.scss and locate the @if branches.
  4. Notice: in compiled CSS, you only see the resulting declarations, not the branching logic.

    You will not see any @if, @else, or unused variant rules in the output. Only the declarations for .badge--hot-lg exist.
