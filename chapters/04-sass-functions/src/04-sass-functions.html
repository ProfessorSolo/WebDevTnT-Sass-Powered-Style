Sass Functions

Functions return values. Think “design math”: consistent spacing, radius, and glow without copy/pasting numbers.

What Sass functions do

  • Functions return values — lengths, colors, numbers. They never emit selectors or rules.
  • Functions centralize design math. Spacing, radius, and glow are computed once and reused consistently.
  • Functions can validate inputs. Invalid arguments fail fast using @error.

Think of functions as pure computation. They answer questions like “how big?” or “how intense?” — not “what selector?”

A simple Sass function

@function space($n) {
  @return $n * 0.25rem;
}

Using the function

.card {
  padding: space(6);
}

Compiled CSS

.card {
  padding: 1.5rem;
}

The functions

Open partials/_functions.scss. These functions return values — not selectors.

  • space(step) → returns a rem-based spacing value from the spacing scale.
  • radius(level) → returns a consistent border-radius value.
  • glow(alpha) → returns an RGBA color used for glow effects.

Function definitions

@function space($step) {
  @return $step * 0.25rem;
}

@function radius($level) {
  @return $level * 0.25rem;
}

@function glow($alpha) {
  @return rgba(0, 255, 255, $alpha);
}

Using the functions

.card {
  padding: space(6);
  border-radius: radius(3);
  box-shadow: 0 0 24px glow(0.35);
}

Each function answers a single question: “What value should I use here?”

The goal isn’t clever math. The goal is one rule, one source of truth, everywhere it’s used.

Same component, different function inputs

All three cards use the same selector. The only difference is the arguments passed to Sass functions.

  • Padding is computed with space()
  • Border radius is computed with radius()
  • Glow intensity is computed with glow()

Inspect a card and jump to the Sass source. You’ll see function calls returning literal values — no mixins, no conditionals.

What to inspect

  1. Inspect .fn-card--tight, .fn-card--base, and .fn-card--roomy. Focus on padding, border-radius, and box-shadow.
  2. In DevTools, click a value (for example padding: 1.5rem) and jump to the Sass source.
  3. Find the function call that produced it (for example padding: fn.space(6)).
  4. Confirm the key idea: compiled CSS contains only literal values. The function calls are gone.
