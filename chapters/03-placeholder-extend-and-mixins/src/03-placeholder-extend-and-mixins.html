Sass · Reuse

Placeholders, @extend, and Mixins

@extend merges selectors. Mixins let you design a tiny styling API: parameters, defaults, and safe reuse.

Two ways to reuse styles

  • Placeholders (%thing) are reusable rule sets. They never generate CSS on their own and only exist to be reused.
  • @extend reuses a placeholder by merging selectors. Multiple selectors end up sharing the same CSS rule.
  • Mixins reuse styles by generating new declarations for each selector that includes them.

Placeholder + @extend

%card-shell {
  border: 1px solid var(--stroke);
  background: var(--surface);
  border-radius: $radius-2;
}

.panel--extend {
  @extend %card-shell;
}

Mixin (parameterized)

@mixin card-shell($radius: $radius-2) {
  border: 1px solid var(--stroke);
  background: var(--surface);
  border-radius: $radius;
}

.panel--mixin {
  @include card-shell($radius: $radius-pill);
}

Placeholders are the raw material. @extend merges selectors. Mixins generate declarations. Same intent, very different output.

@extend (selector merge)

Two different contexts extend the same placeholder. In compiled CSS you’ll see a merged selector list. That list is the coupling.

Sass source

%btn-base {
  padding: $space-2 $space-4;
  border-radius: $radius-pill;
}

.zone-alpha .btn {
  @extend %btn-base;
}

.zone-beta .btn {
  @extend %btn-base;
}

Compiled CSS

.zone-alpha .btn,
.zone-beta .btn {
  padding: 0.5rem 1rem;
  border-radius: 999px;
}

In Sass: both buttons @extend %btn-base from different parent selectors. In CSS: those selectors get merged into one rule.

Any future change to %btn-base now affects both contexts at once, even if they were never meant to be related.

The mixin

@mixin badge(
  $size: md,
  $tone: cool,
  $glow: false
) {
  padding: map-get($sizes, $size);
  background: map-get($tones, $tone);

  @if $glow {
    box-shadow: 0 0 24px map-get($glows, $tone);
  }
}

Variant wrappers

.badge--sm   { @include badge(sm); }
.badge--md   { @include badge(md); }
.badge--lg   { @include badge(lg); }

.badge--hot  { @include badge(md, hot); }
.badge--glow { @include badge(md, cool, true); }

Each class includes the same mixin with different arguments. The output CSS is duplicated by design — but it stays local, explicit, and easy to reason about.

What to inspect

  1. In the @extend tab, inspect .btn--alpha and .btn--beta. In the compiled CSS, find the merged selector list.
  2. Notice that both selectors now point to the same rule, even though they live in different parts of the UI.
  3. Switch to the Mixins tab and inspect .m-btn--cool, .m-btn--hot, and .m-btn--ghost.
  4. Notice that each variant produces its own rule. No cross-context coupling. No unexpected selector merges.
  5. Find the breakpoint mixin and observe how it replaces repeated media-query scaffolding with a single abstraction.
