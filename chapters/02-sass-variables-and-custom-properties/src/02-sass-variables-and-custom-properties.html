Sass Variables vs CSS Custom Properties

Sass variables compile away. CSS custom properties stick around and can change at runtime.

Sass variables compile away. CSS variables persist.

Sass variables and CSS custom properties solve different problems. This demo shows where each one lives in the lifecycle of your styles.

  • Sass variables are resolved at build time and do not exist in the browser.
  • CSS custom properties exist at runtime and can be changed without recompiling.
  • The right choice depends on when a value needs to change.

Sass variables (build time)

// tokens.scss
$space-6: 1.5rem;
$radius-2: 0.75rem;

.card {
  padding: $space-6;
  border-radius: $radius-2;
}

Compiled CSS output

.card {
  padding: 1.5rem;
  border-radius: 0.75rem;
}

CSS custom properties (runtime)

:root {
  --surface: rgba(20, 30, 50, 0.6);
  --stroke: rgba(255, 255, 255, 0.2);
}

.card {
  background: var(--surface);
  border-color: var(--stroke);
}

Sass variables shape the CSS you generate. CSS custom properties remain live and can respond to user interaction.

Mental model

Sass variables and CSS custom properties exist at different points in time. Understanding when they exist determines which one you should use.

Sass variables are compile-time only. They help you author CSS, but they do not exist in the browser. In the compiled output, only the final literal values remain.

CSS custom properties are runtime values. They live in the DOM, participate in the cascade and inheritance, and can be changed after the page loads.

Decision rule

Use Sass variables when the value is fixed at build time.
Use CSS custom properties when the value must change at runtime.

Most real projects use both: Sass for structure and math, CSS custom properties for themes and user-driven changes.

What to inspect

This step highlights the practical difference between Sass variables and CSS custom properties in the browser.

  1. Inspect the .token-card in the Demo tab.
  2. In the Styles pane, find the background and border rules. Notice they use var(--surface) and var(--stroke).
  3. Click the rule source to jump to main.scss or a partial.
  4. Toggle the theme and watch the computed values change without any Sass recompilation.

Search for these in DevTools

--surface      ✔ present (runtime)
--stroke       ✔ present (runtime)
$space-6       ✖ not present (compiled away)

In the Computed panel, search for --surface. You should see it resolved from :root or [data-theme="neon"], depending on the active theme.
